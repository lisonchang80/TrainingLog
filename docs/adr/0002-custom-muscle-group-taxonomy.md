# 0002 — 自訂 11-MG 分類，拒絕 Standard 模板

採用自訂的 11 個 Muscle Group 分類，而非常見的 Lite (7) / Standard (10) / Detailed (14) 模板。

最終 MG 列表（11 個）：胸、背、腿、臀、肩、斜方肌、二頭、三頭、小腿、前臂、核心。

> **已被 ADR-0017 Q9 取代**（overnight audit 2026-05-30 註記）：MG「前臂」經 ADR-0017 Q9 + migration v010 更名為「**小臂**」；上列為歷史原始命名，現行命名以 ADR-0017 / ADR-0010 § 2026-05-13 amendment 為準。

關鍵的非主流選擇：

1. **背用「功能」切，不用「解剖位置」切**：Sub-Group 為「水平 / 垂直」（拉的方向），
   而非常見的「上背 / 闊背 / 下背」。理由：訓練分化日常以動作平面決定選擇（划船 vs 引體），
   解剖位置切會讓同一動作（例：T-bar row）橫跨多個 SG，難以乾淨 tag。

2. **刪除「腹部」MG，全部歸入「核心」**：避免「腹部 ⊂ 核心」的階層歧義。
   所有腹直肌/腹斜的直接訓練（卷腹、leg raise）與抗旋/抗伸穩定訓練（Pallof press, dead bug）
   一律 tag 為 MG=核心。

3. **「臀」獨立 MG**（含上臀/下臀 SG），不併入腿：使用者重視臀部訓練細節，獨立才能讓
   Hip Thrust、Glute Bridge、Cable Kickback 等動作有正確歸屬。

4. **腿只切「腿前/腿後」兩 SG**，不三切「股四頭/腿後/小腿」：使用者實際分化習慣以
   前後鏈為主；「小腿」獨立成 MG（不在腿之下）。

5. **「斜方肌」獨立 MG**：聳肩、Face Pull 等動作頻繁但不適合塞進背或肩。

—— 拒絕的替代方案：

- **Standard (10)**：股四頭 / 腿後 / 臀 / 小腿 全部當 MG。原因：使用者不需要這層細度，且
  「腿後/臀」這種雙頭命名違反 MG 應該單一概念的原則。
- **Lite (7) + 完整 SG**（最初提案的「跟訓記一樣」）：訓記式 7 MG 把二頭三頭塞進「手臂」MG，
  使用者實際訓練分化常獨立練二頭/三頭，所以升上來當 MG 更好用。

修改此分類的代價：每個 Exercise 的 MG/SG tag 都要重打，且歷史 Session 的統計分析會斷層。
