# 新增一支內建動作到 Exercise Library

## 先分清楚兩條路（別走錯）

- **Runtime 自訂**（`createCustomExercise` / `app/exercise/new.tsx`、`is_custom=1`）＝使用者在 app 內自己按「新增動作」。使用者叫「幫我加動作」時**不是**這條。
- **Seed 內建庫**（migration 帶、`is_builtin=1 is_custom=0`）＝要你（開發者）加的就是這條。以下全指這條。

## 檔頭警告：generated 檔勿手改

`src/db/seed/v028ExerciseMediaLibrary.ts` / `v029ExerciseMuscleLinks.ts` / `exerciseMediaMap.ts` 檔頭寫明「GENERATED by build_seed.py，勿手改」。**加一支動作不要重跑整條 python 管線**（會重生 230+ 動作、diff 很吵、還要 fetch FE-DB）。改走 **新 migration delta**（自足 INSERT），generated 檔保持 frozen。

## Recipe（單支動作，無真人照 placeholder 版）

實證 2026-07-07 加「跪姿滑輪下拉」（`v030`、id `...000111`）。步驟：

### 1. 找下一個空 id + 現行 migration tip
```bash
# 真實最大 id（含 3-hex 尾）；下一個 = +1
grep -oE "8000-0000000[0-9a-f]{3}'" src/db/seed/v028ExerciseMediaLibrary.ts | sort | tail -1
# 現行最高 migration（新的 = +1）
ls -1 src/db/schema/ | sort | tail -1
```
v028 用 exId(67..272)＝id `...110`。下一個空的＝`...111`（273）。

### 2. 新 migration `src/db/schema/v0NN_<slug>.ts`
自足 INSERT OR IGNORE（idempotent），鏡像 v028 的欄位順序：
```ts
await db.runAsync(
  `INSERT OR IGNORE INTO exercise
     (id, name, load_type, is_builtin, is_custom, muscle_group_id, equipment, media_path)
   VALUES (?, ?, ?, 1, 0, ?, ?, ?)`,
  EXERCISE_ID, '跪姿滑輪下拉', 'loaded', 'mg-back', '滑輪', null,   // media_path null = placeholder hash-color thumb（合法既有型態）
);
// 肌肉高亮：primary + secondary（鏡像同家族既有動作的 v029 連結）
for (const l of MUSCLE_LINKS)  // {muscle_id, role}
  await db.runAsync(`INSERT OR IGNORE INTO exercise_muscle (exercise_id, muscle_id, role) VALUES (?,?,?)`, EXERCISE_ID, l.muscle_id, l.role);
```
- `equipment` 8-enum：槓鈴/啞鈴/史密斯機/滑輪/固定機械/自重/壺鈴/其他。
- `muscle_group_id`：mg-back/mg-chest/mg-leg/…；19 肌肉 id 見 `v006ExerciseLibrary.ts`（m-back/m-bicep-long/m-bicep-short/m-tricep/…）。
- 找**同家族既有動作**（如「對握滑輪下拉」）鏡像 muscle_group/equipment/muscle links。

### 3. 註冊 `src/db/migrate.ts`
import + 加 `NN: v0NN_<slug>,` 到 `migrations` map 尾。

### 4. i18n `src/i18n/strings.ts`（zh + en 都要）
DB `name` 存中文；zh.exercise 加 `'跪姿滑輪下拉': '跪姿滑輪下拉'`、en.exercise 加英文名。插在同家族附近（`grep -n "滑輪下拉'" src/i18n/strings.ts` 找位置）。**placeholder（無照）entries 在各自 sorted 區塊**，real（有照）在 seed order 區塊。

### 5. 防未來 re-gen drift（source of truth）
- `docs/exercise-media-import/curated-master.json`：**append 到陣列尾端**（build_seed.py 從 `next_id=67` 依**檔案順序**指派 id，append 尾端才穩拿下一個 id、中間插會 renumber）。placeholder 格式：`en:null, id:null, images:null, img:"placeholder"`。
- `build_seed.py` `PLACEHOLDER_EN` dict：加英文名（否則未來 re-gen 會 `assert not missing_en` 爆掉）。

### 6. ⚠️ 測試計數斷言（最大地雷）
加 1 支 active builtin ⇒ 撞**散在多檔的硬編計數**。naive grep 只看 head 會漏。全掃：
```bash
grep -rnE "\b233\b|\b272\b" tests/          # active 233→234；is_builtin 272→273
grep -rn "toBe(29)" tests/db/migrateChain.test.ts   # max version 29→30（每加一支 migration 都要）
```
2026-07-07 實測需改（233 基準時）：
- **active 233→234**：`exerciseLibrary.test.ts`(×3)、`templates.test.ts`、`sessionLifecycle.test.ts`(×2)、`setRepository.test.ts`(×2)、`v028_exercise_media_library.test.ts`(×1)
- **is_builtin 272→273**：`v028_exercise_media_library.test.ts`(×2 + it-name)
- **max user_version 29→30**：`migrateChain.test.ts`(×4 + comment + it-name)；`v029_*.test.ts` 是 `toBeGreaterThanOrEqual` 不用改；`restoreService.test.ts` 用動態 `migrationsMaxVersion()` 不用改；`closeAndResetForRestore*.test.ts` 是 mock 值不用改。

### 7. 加一支聚焦回歸測試
在 `tests/db/exerciseLibrary.test.ts` 用 `getExerciseWithMuscles(db, id)` 斷言新動作存在 + muscle_group + equipment + media_path null + primary/secondary links。

### 8. Gate
`npx tsc --noEmit` + `npx jest`（全綠，數字 = 舊 baseline +1 支新測試）。

## 為何 delta migration 而非重跑 generator
- 尊重 generated 檔「勿手改」檔頭；diff 小、可 review。
- 免 FE-DB fetch / 免整庫重生。
- 未來若有人真的重跑 generator：curated-master 已有這支 → 併回 v028、delta migration 變 idempotent no-op（INSERT OR IGNORE）。
- 相關：`migration-new` skill（一般 migration 慣例）。
